# Polymarket 自动赎回工具项目计划表

本文件用于跟踪项目的开发进度。

## 项目阶段概览

- [x] **第 1 段：查询持仓，只筛选 redeemable**
    - **目标**：零风险验证数据链路。
    - **功能**：读取地址配置，查询 Polymarket 当前持仓，过滤 `redeemable = true`。
    - **状态**：✅ 已完成（包含 `size >= 0.1` 过滤逻辑）。

- [x] **第 2 段：对接 ServerChan 通知系统**
    - **目标**：打通消息推送链路与基础去重。
    - **功能**：集成 ServerChan / FTQQ Push 接口，实现 `conditionId + outcome` 唯一标识去重，本地 `state.json` 持久化。
    - **状态**：✅ 已完成（支持合并通知，具备失败重试机制）。

- [ ] **第 3 段：手动触发 redeem**
    - **目标**：验证资金赎回操作的核心逻辑。
    - **功能**：指定 `conditionId` 手动执行 redeem 操作，成功后发送通知并记录。
    - **测试标准**：手动指定一笔已结算仓位，脚本能成功 redeem 且收到成功消息。

- [ ] **第 4 段：自动批量 redeem**
    - **目标**：实现全自动无人值守赎回。
    - **功能**：自动扫描全部符合条件的仓位，逐个执行 redeem，记录并分别通知成功/失败结果。
    - **核心点**：幂等保护，确保同一笔不重复处理。

- [ ] **第 5 段：增加轮询，变为常驻脚本**
    - **目标**：从“手动触发”升级为“后台监控”。
    - **功能**：设置定时轮询（如每 5 分钟），增加异常捕获和日志管理，支持优雅退出。

- [ ] **第 6 段：生产环境部署与优化**
    - **目标**：长期稳定运行。
    - **功能**：使用 PM2 部署，日志管理，可选将 JSON 存储升级为 SQLite。
